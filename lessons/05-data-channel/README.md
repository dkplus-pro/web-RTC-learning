# 第 05 课：RTCDataChannel 文本聊天

本课从音视频扩展到点对点数据传输。你会创建一个 `RTCDataChannel`，在同一页面模拟两个端点收发文本消息，并观察通道打开、发送、接收与关闭的完整生命周期。

## 学习目标

- 理解 `RTCDataChannel` 适合低延迟点对点数据，而不是替代所有客户端/服务器通信。
- 掌握在 offer 端创建数据通道、在 answer 端通过 `datachannel` 事件接收通道的流程。
- 认识可靠、有序传输与低延迟之间的配置取舍。
- 学会用连接状态和通道状态判断消息是否可以发送。

## 关键概念

### DataChannel 复用 WebRTC 传输能力

数据通道建立在 WebRTC 的安全传输之上，通常使用 SCTP over DTLS。只要点对点连接成功，就可以在无需额外 WebSocket 消息中转的情况下交换数据。

### 聊天只是最小示例

实际业务可以把数据通道用于协同光标、白板操作、游戏状态、小文件分片或旁路控制消息。但它不适合需要服务端持久化、审计或全量广播的消息流。

## 运行方式

```bash
npm run lesson:05
# 浏览器打开 http://localhost:5105
```

## 实验步骤

1. 点击“建立数据通道”，等待两端通道状态变为 `open`。
2. 在左侧输入消息并发送，观察右侧收到的内容。
3. 在右侧回复消息，确认双向通信可用。
4. 关闭连接后再次发送，观察界面对不可发送状态的保护。

## 架构提醒

生产环境中，数据通道通常与服务端信令、权限系统和业务状态同步一起使用。需要明确哪些数据必须经过服务端，哪些数据可以点对点直连，以避免审计缺口和一致性问题。
